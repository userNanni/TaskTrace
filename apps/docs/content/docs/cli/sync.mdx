## Synopsis

```bash
tt sync [options]
tasktrace sync [options]
```

## Description

Executes the sync plan generated by `tt plan`. Sends pending `WorklogEntry` records to the
configured task provider (e.g. ClickUp). Updates local sync state to prevent duplicates.

## Options

| Flag | Alias | Type | Default | Description |
|---|---|---|---|---|
| `--entry` | `-e` | `string` | — | Sync a single entry by ID |
| `--dry-run` | | `boolean` | `false` | Simulate without writing to provider |
| `--json` | | `boolean` | `false` | Output as JSON |

## Examples

```bash
# Sync all pending entries
tt sync

# Dry-run (same as tt plan but via sync command)
tt sync --dry-run

# Sync a single entry
tt sync --entry 01JRZXABCDEFGH123456789000

# JSON output
tt sync --json
```

```

```

## What sync does per mode

### `comment` mode

Posts a Markdown comment to the linked ClickUp task:

```markdown
## Work log — 2026-04-14

**Summary:** Fixed auth timeout and refresh token flow
**Duration:** 45 min | **Kind:** bugfix | **Branch:** CU-abc123/fix-auth

**Commits:**
- abc123def456

**Tags:** auth, tokens
```

### `subtask` mode

Creates a new subtask under the linked parent task with the entry summary as its name
and the formatted Markdown as its description.

### `task` mode

Creates a brand-new task in the configured `defaultListId`. Used when there is no task reference
on the entry (e.g. exploratory work not tied to a ticket).

## Sync state

After a successful sync, TaskTrace writes to `.tasktrace/sync-state.json`:

```json
{
  "lastSyncAt": "2026-04-14T17:00:00.000Z",
  "entries": {
    "01JRZXABCDEFGH123456789000": {
      "syncStatus": "synced",
      "syncMode": "comment",
      "providerRef": "CU-abc123",
      "syncedAt": "2026-04-14T17:00:00.000Z"
    }
  }
}
```

This prevents the same entry from being synced twice.

<Callout type="warn">
  Sync requires a configured provider. Make sure your API key is set in
  `~/.tasktrace/config.json` or via `TASKTRACE_CLICKUP_API_KEY` before running `tt sync`.
  Run `tt plan` first to verify the payload.
</Callout>

## Error handling

If a sync fails for one entry, other entries still proceed. Failed entries are marked
`syncStatus: failed` in sync state and can be retried:

```bash
tt sync --entry <failed-entry-id>
```
