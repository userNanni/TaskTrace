## Synopsis

```bash
tt plan [options]
tasktrace plan [options]
```

## Description

Generates a dry-run preview of what `tt sync` would send. No data is written — neither locally
nor to the task provider. Use this to review and approve before committing.

**Always run `tt plan` before `tt sync`** in unfamiliar situations.

## Options

| Flag | Type | Default | Description |
|---|---|---|---|
| `--json` | `boolean` | `false` | Output as JSON |

## Examples

```bash
# Preview sync plan
tt plan

# JSON output (for AI agents or scripts)
tt plan --json
```

## Sync modes in the plan

| Mode | Condition | Action |
|---|---|---|
| `comment` | Entry has `taskRef` + short session | Adds comment to existing task |
| `subtask` | Entry has `taskRef` + long session or many commits | Creates subtask under parent |
| `task` | No `taskRef` | Creates new task in `defaultListId` |
| `skip` | Explicitly excluded | No action taken |

<Callout type="info">
  The adapter auto-suggests a sync mode based on duration and commit count. You can override
  this in your `.tasktracerc.json` via `defaults.syncMode`.
</Callout>

## Typical workflow

```bash
tt consolidate   # group pending events
tt attach        # link commits
tt plan          # review — make sure actions look right
tt sync          # execute
```
