## Configuration hierarchy

Settings are merged in this order (last wins):

```
defaults (built-in)
  ↓
~/.tasktrace/config.json   (global, per-user)
  ↓
.tasktracerc.json          (project, version-controlled)
  ↓
env vars                   (CI, shell)
  ↓
CLI flags                  (per-command)
```

<Callout type="warn">
  **Credentials never go in project config.** API keys must live in
  `~/.tasktrace/config.json` or environment variables.
</Callout>

## Project config `.tasktracerc.json`

Generated by `tt init`. Commit this file — it defines provider and git settings for your team.

```json title=".tasktracerc.json"
{
  "version": 1,
  "storage": {
    "dir": ".tasktrace"
  },
  "git": {
    "branchPatterns": [
      "CU-([a-zA-Z0-9]+)",
      "([A-Z]+-\\d+)",
      "#(\\d+)"
    ],
    "hooks": {
      "preCommit": false,
      "postCommit": true
    }
  },
  "provider": {
    "name": "clickup",
    "config": {
      "defaultListId": "12345678"
    }
  },
  "defaults": {
    "syncMode": "comment",
    "kind": "implementation",
    "consolidationGapMinutes": 240
  }
}
```

### `storage`

| Field | Type | Default | Description |
|---|---|---|---|
| `dir` | `string` | `.tasktrace` | Storage directory path (relative to project root) |

### `git`

| Field | Type | Default | Description |
|---|---|---|---|
| `branchPatterns` | `string[]` | `["CU-([a-zA-Z0-9]+)", "([A-Z]+-\\d+)", "#(\\d+)"]` | Regex patterns for extracting task refs from branch names |
| `hooks.preCommit` | `boolean` | `false` | Install pre-commit hook |
| `hooks.postCommit` | `boolean` | `false` | Install post-commit hook (auto-attach commit) |

**Branch pattern examples:**

| Branch name | Pattern | Extracted ref |
|---|---|---|
| `CU-abc123/fix-auth` | `CU-([a-zA-Z0-9]+)` | `CU-abc123` |
| `feature/PROJ-42-add-login` | `([A-Z]+-\\d+)` | `PROJ-42` |
| `fix/#99-crash-on-load` | `#(\\d+)` | `#99` |

### `provider`

| Field | Type | Description |
|---|---|---|
| `name` | `"clickup"` | Task provider name |
| `config.defaultListId` | `string` | ClickUp list ID for new tasks |
| `config.workspaceId` | `string?` | ClickUp workspace ID (optional) |
| `config.assignees` | `number[]?` | Default assignee user IDs (optional) |

### `defaults`

| Field | Type | Default | Description |
|---|---|---|---|
| `syncMode` | `"comment" \| "subtask" \| "task" \| "skip"` | `"comment"` | Default sync strategy |
| `kind` | `WorklogKind` | `"implementation"` | Default event kind when `-k` is omitted |
| `consolidationGapMinutes` | `number` | `240` | Minutes of silence that split a session into separate entries |

## Global config `~/.tasktrace/config.json`

User-level config. Never committed. Store credentials here.

```json title="~/.tasktrace/config.json"
{
  "version": 1,
  "providers": {
    "clickup": {
      "apiKey": "pk_your_clickup_api_key"
    }
  }
}
```

## Environment variables

<Tabs items={["ClickUp", "Storage", "General"]}>
<Tab value="ClickUp">

| Variable | Description |
|---|---|
| `TASKTRACE_CLICKUP_API_KEY` | ClickUp API key — overrides global config |
| `TASKTRACE_CLICKUP_LIST_ID` | Default list ID — overrides project config |

</Tab>
<Tab value="Storage">

| Variable | Description |
|---|---|
| `TASKTRACE_STORAGE_DIR` | Override storage directory path |
| `TASKTRACE_PROJECT_DIR` | Override project root (used by MCP server) |

</Tab>
<Tab value="General">

| Variable | Description |
|---|---|
| `TASKTRACE_LOG_LEVEL` | Log verbosity (`debug`, `info`, `warn`, `error`) |

</Tab>
</Tabs>

## CLI flag overrides

Any CLI flag takes precedence over all other config. Example:

```bash
# Use a different storage dir just for this command
tt status --dir /tmp/my-project

# Override sync mode for a single sync
tt sync --mode subtask
```

## Full defaults reference

Built-in defaults (from `@tasktrace/core`):

```typescript
{
  version: 1,
  storage: {
    dir: ".tasktrace",
  },
  git: {
    branchPatterns: [
      "CU-([a-zA-Z0-9]+)",
      "([A-Z]+-\\d+)",
      "#(\\d+)",
    ],
    hooks: {
      preCommit: false,
      postCommit: false,
    },
  },
  defaults: {
    syncMode: "comment",
    kind: "implementation",
    consolidationGapMinutes: 240,
  },
}
```

## ClickUp API key setup

<Tabs items={["Global config (recommended)", "Environment variable", "CI/CD"]}>
<Tab value="Global config (recommended)">

```bash
mkdir -p ~/.tasktrace
cat > ~/.tasktrace/config.json <<'EOF'
{
  "version": 1,
  "providers": {
    "clickup": {
      "apiKey": "pk_YOUR_API_KEY_HERE"
    }
  }
}
EOF
```

</Tab>
<Tab value="Environment variable">

```bash
export TASKTRACE_CLICKUP_API_KEY=pk_YOUR_API_KEY_HERE
```

Add to `~/.zshrc` or `~/.bashrc` for persistence.

</Tab>
<Tab value="CI/CD">

Set `TASKTRACE_CLICKUP_API_KEY` as a secret in your CI environment. Do not hardcode keys in any config file.

</Tab>
</Tabs>

Get your ClickUp API key: **ClickUp Settings → Apps → API Token**.

## Finding your ClickUp list ID

1. Open ClickUp in a browser
2. Navigate to the list you want to sync to
3. Copy the list ID from the URL: `app.clickup.com/123456/v/li/**901234567**`

Set it in `.tasktracerc.json`:

```json
{
  "provider": {
    "name": "clickup",
    "config": {
      "defaultListId": "901234567"
    }
  }
}
```
