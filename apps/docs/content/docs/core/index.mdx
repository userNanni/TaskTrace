## Overview

`@tasktrace/core` is the foundation of TaskTrace. All other packages depend on it; it has
**zero intra-monorepo dependencies**.

```
@tasktrace/cli
@tasktrace/mcp-server        →  @tasktrace/core
@tasktrace/adapter-clickup
```

## Runtime compatibility

Core targets **Bun-native** execution with Node.js compatibility via conditional exports:

```json
{
  "exports": {
    ".": {
      "bun": "./src/index.ts",
      "default": "./dist/index.js"
    }
  }
}
```

This means:
- Bun: runs TypeScript directly, no build step needed in development
- Node.js: requires `dist/` build output

## Internal dependencies

| Dependency | Purpose |
|---|---|
| `ulid` | Sortable, collision-free IDs |
| `zod` | Runtime schema validation at all data boundaries |
| `proper-lockfile` | Concurrent-safe NDJSON writes |

No HTTP clients, no LLM SDKs, no external APIs.
