## What is an adapter?

An adapter implements the `TaskProviderAdapter` interface from `@tasktrace/core`, bridging
TaskTrace's internal `WorklogEntry` format to a specific task manager's API.

```typescript
interface TaskProviderAdapter {
  testConnection(): Promise<{ ok: boolean; error?: string }>;
  resolveTask(taskRef: string): Promise<ProviderTask | null>;
  createTask(params: CreateTaskParams): Promise<ProviderTask>;
  syncEntry(entry: WorklogEntry, mode: SyncMode): Promise<SyncResult>;
  suggestSyncMode(entry: WorklogEntry): SyncMode;
}
```

Any package that implements this interface can be plugged into TaskTrace as a provider.

## Writing a custom adapter

1. Create a package that imports `@tasktrace/core`
2. Implement `TaskProviderAdapter`
3. Configure it in `.tasktracerc.json` via `provider.name`

```typescript
import type { TaskProviderAdapter, WorklogEntry, SyncMode, SyncResult } from "@tasktrace/core";

export class MyAdapter implements TaskProviderAdapter {
  async testConnection() {
    // verify credentials
    return { ok: true };
  }

  async resolveTask(taskRef: string) {
    // fetch task by ref from your API
    return null;
  }

  async createTask(params) {
    // create a new task
    return { id: "...", name: params.name, url: "..." };
  }

  async syncEntry(entry: WorklogEntry, mode: SyncMode): Promise<SyncResult> {
    // post comment / create subtask / create task
    return { ok: true, providerRef: "...", url: "..." };
  }

  suggestSyncMode(entry: WorklogEntry): SyncMode {
    return entry.taskRef ? "comment" : "task";
  }
}
```

<Callout type="info">
  Adapters are loaded lazily — only when sync commands are used. The core engine and CLI work
  without any adapter configured.
</Callout>
